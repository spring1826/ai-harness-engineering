1편 하네스란?

하네스 엔지니어링이란?

AI는 힘이 세고 빠른 말이에요.

질문하면 바로 답하고,
복잡한 일도 순식간에 처리해요.

하지만 고삐 없이 풀어놓으면
제멋대로 달려요.

같은 질문을 해도 어제는 3줄, 오늘은 10줄.

어제는 표로 정리했는데, 오늘은 글로 풀어써요.

팀원 A에게는 격식체, 팀원 B에게는 반말.

이러면 업무에 쓸 수 없어요.

실제로 이런 일이 생겨요

마케팅팀이 AI에게 광고 문구를 요청했는데, 매번 형식이 달라서 그대로 쓸 수 없었어요.
→ 대체 편차 방지
AI를 켤 때마다 "어제 뭐 했냐면..." 매번 설명해야 했어요.
→ 부적절한 엔지니어링 방지
요청을 접수할 때 빠뜨리는 항목이 매번 달랐어요.
→ 불완전한 작업 완료 방지
AI에게 "이거 해도 될까?" 물어보면 항상 "좋습니다"만 해요. 위험한 부분은 알아서 안 찾아줘요.
→ 부적절한 테스트 방지

이 모든 문제의 원인은 하나예요.
말에 마구를 채우지 않았기 때문이에요.

하네스 엔지니어링

AI에 마구(하네스)를 채워서,
우리가 원하는 방향으로,
원하는 속도로 달리게 만드는 것

"하네스(harness)"는 실제로 말에 채우는 마구에서 온 용어예요.
고삐, 재갈, 안장, 눈가리개 — 이것들을 합쳐서 하네스라고 불러요.

이 개념은 HashiCorp 창업자 Mitchell Hashimoto가 처음 이름 붙였어요.
"AI가 실수하면, 그 실수가 다시 생기지 않도록 환경을 만드는 것."
지금은 OpenAI, 토스 등 여러 회사에서 사용하는 방법론이 되었어요.

AI에게 채우는 마구 4가지

🪢
고삐 (규칙서)
기본 방향과 규칙을 정해요
"왼쪽으로", "천천히"
→ CLAUDE.md
📋
마구 점검 (자동 브리핑)
출발할 때 자동으로 상황을 파악해요
매번 출발 전 자동 체크
→ Hooks
🗺️
경로 (세트 명령)
한마디면 정해진 길로 달려요
"A코스!" 한마디로 끝
→ Skills
🧑‍✈️
전문 마부 (전문가 역할)
관점별 전문가를 배정해요
산길 전문, 야간 전문
→ Agents

이 4가지를 하나씩 갖추면,
누가 AI에게 질문해도 매번 같은 품질의 답이 나와요.

한 번에 완성이 아니라 과정이에요

마구를 한 번에 완벽하게 채울 필요는 없어요.
고삐 3줄로 시작해서, 달리면서 하나씩 다듬어가면 돼요.

말이 오른쪽으로 치우치면 고삐를 조정하고,
속도가 안 맞으면 재갈을 조이듯이.

쓰면서 다듬을수록, 나만의 맞춤형 AI가 만들어져요.

이 시리즈에서 사용하는 도구

Claude Code

대화하듯 AI에게 지시하는 도구예요.

여기에 마구를 채우는 방법을
이 시리즈에서 하나씩 알려드릴게요.

직접 만들 필요 없이,
AI에게 "이런 마구를 만들어줘"라고 말하면 돼요.
